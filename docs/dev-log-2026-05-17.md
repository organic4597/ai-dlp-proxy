# AI DLP Proxy — 2026-05-17 패치 일지

## 한눈에 요약
오늘은 "탐지는 되는데 대시보드에 안 보이는 문제", "sLM이 실제로 호출되는지 불명확한 문제", "탐지 목록 상세 확인 기능 회귀"를 중심으로 안정화 작업을 진행했다.

- 엔진/웹 구독 경로 안정화 및 이벤트 누락 이슈 완화
- 프롬프트 문맥(prompt excerpt) 저장/노출 경로 전면 연결
- sLM API 연동 검증 및 호환성 fallback 추가
- 탐지 목록 탭 상세 보기(행 클릭) 기능 복구
- 외부 sLM 서버 저장소 작업 내역 통합 정리

---

## 작업 배경
운영 중 다음 증상이 반복됐다.

1. Windows OpenCode에서 입력해도 트래픽/탐지가 대시보드에 비어 보이는 구간이 발생
2. sLM 파이프라인이 실제로 호출되는지 확인이 어려움
3. 탐지 목록 탭에서 항목 클릭 시 상세가 열리던 기능이 보이지 않음

핵심 목표는 "실제 탐지 이벤트가 어디서 끊기는지 가시화"하고, "API/엔진/웹/UI가 동일한 요청 ID 기준으로 일관되게 보이도록" 만드는 것이었다.

---

## 1) 탐지 파이프라인 판별/정책 안정화

### 변경 포인트
- 공급자(provider) 판별에서 IP 대역 fallback 의존을 줄이고, SNI/Host/URL/Body 기반 판별을 강화
- 정책 적용 시 effective finding(임계값 이상, 비억제, 비history) 기준으로 block/mask가 일관되게 계산되도록 가드 보강

### 관련 파일
- src/engine/extractor.py
- scripts/inspect_traffic.py

### 효과
- "effective=0인데 blocked처럼 보이는" 혼선을 줄이고, 정책 판정의 설명 가능성이 좋아짐

---

## 2) 엔진 이벤트 브리지 안정화

### 변경 포인트
- scan 결과만 브로드캐스트되도록 정리해 비-scan 액션 오염 차단
- scan 처리 구간을 별도 스레드 오프로딩 + lock 직렬화로 이벤트 루프 블로킹 완화
- 웹 구독 클라이언트의 ACK 대기/연결 타임아웃 튜닝 및 idle read 동작 개선

### 관련 파일
- scripts/engine_server.py
- web/backend/engine_client.py

### 효과
- 구독 재연결 경고 루프 빈도 감소
- 장시간 스캔 이후에도 이벤트/트래픽 적재 연속성 개선

---

## 3) 프롬프트 문맥 가시성 강화

### 변경 포인트
- requests 테이블에 prompt_excerpt 저장 경로 추가
- findings/traffic API에 prompt_excerpt 포함
- 프론트 목록/상세에서 프롬프트 문맥 컬럼 노출

### 관련 파일
- web/backend/db.py
- web/backend/models.py
- web/backend/routers/findings.py
- web/frontend/src/routes/traffic/+page.svelte
- web/frontend/src/routes/findings/+page.svelte

### 효과
- "무엇을 감지했는지"뿐 아니라 "어떤 문맥에서 감지됐는지"를 UI에서 즉시 확인 가능

---

## 4) sLM API 연동 검증과 호환성 패치

### 배경
라즈베리파이 경유 sLM API(http://localhost:8766) 자체는 응답하지만, 엔진 경유 scan 경로에서는 특정 payload 조합에서 500이 발생하는 케이스가 확인됐다.

### 원인 요약
- 다중 텍스트 + prior_ranges_per_text 조합에서 원격 API 구현 버전별 호환성 차이 존재

### 변경 포인트
- API 요청 시 prior_ranges_per_text가 실질적으로 비어 있으면 필드 생략
- 우선 요청 실패 시 prior_ranges_per_text 제외 요청으로 1회 fallback 재시도

### 관련 파일
- src/engine/pipeline/slm_stage.py

### 효과
- 엔진 scan 경로에서 backend=api 집계로 정상 호출 확인
- API 버전 차이로 인한 500의 운영 영향 완화

---

## 5) 탐지 목록 상세 보기 기능 복구

### 배경
탐지 목록 탭에서 행 클릭 상세 패널이 회귀되어 사라진 상태였다.

### 변경 포인트
- 행 클릭 선택/해제 로직 추가
- 선택 행 하이라이트 추가
- 우측 상세 패널(요청 메타, 판정 사유, 매치 원문, 동일 요청 탐지 목록) 복구
- request_id 기준 상세 조회 연결

### 관련 파일
- web/frontend/src/routes/findings/+page.svelte

### 효과
- 탐지 목록에서 즉시 원인 분석 가능
- traffic 탭으로 이동하지 않아도 1차 분석이 가능해짐

## 6) 운영 설정/헬스체크 UX 개선

### 변경 포인트
- 웹 백엔드 정적 서빙을 SPA fallback 방식으로 보강
- 인증서 디렉터리(/certs) 정적 서빙 추가
- SLM 상태 확인용 API endpoint 추가(/api/slm/health)
- Settings 화면에서 SLM backend(auto/gguf/adapter/api), API URL 편집, health 확인 UI 추가
- Process 화면에서 start/stop 직후 즉시 상태 오판을 줄이기 위한 지연 재조회 추가

### 관련 파일
- web/backend/main.py
- web/backend/routers/control.py
- web/backend/routers/pipeline.py
- web/frontend/src/lib/api.ts
- web/frontend/src/routes/settings/+page.svelte
- web/frontend/src/routes/process/+page.svelte

### 효과
- 운영자가 "SLM이 실제 연결됐는지"를 UI에서 즉시 판단 가능
- 프로세스 시작/중지 직후 상태 반영 신뢰도 향상

## 7) 외부 sLM 서버 저장소 작업(사용자 제공 내역 반영)

아래 항목은 외부 저장소(qwen_tunning)에서 같은 날짜에 진행된 작업으로, 본 패치 일지에 통합 반영한다.

1. SLM API 서버 관측성 강화
- detect/detect_batch 상세 로그
- 느린 요청 경고 기준
- 누적 통계 endpoint 강화

2. detect_batch 500 호환성 수정
- prior_ranges_per_text의 빈 항목 처리 안정화

3. 긴 입력 성능 최적화
- PII 힌트 없음 텍스트 skip
- 첫 탐지 청크에서 조기 종료(early stop)
- skipped_chunks, early_stops 통계 반영

4. 오탐 완화
- placeholder/라벨 문자열(api_key_assignment 류) 파싱 단계 필터링

5. 실동작 검증
- detect_batch + 빈 prior_ranges_per_text 정상 통과
- 긴 텍스트 조기 종료 통계 증가 확인
- placeholder 오탐 억제 + 정상 이메일 탐지 유지 확인

## 검증 결과 요약

1. 엔진 경유 장시간 스캔
- stress-scan 케이스에서 100초대 처리 후 traffic 적재 확인

2. 정책 반영 검증
- alert 요청에 applied_result=blocked 반영 시 DB row에 dlp_applied=blocked 확인

3. sLM API 직접 검증
- /health, /detect, /detect_batch, /stats 응답 확인
- payload 조합에 따른 500 재현 및 fallback 적용 후 엔진 경유 정상화 확인

4. 프론트 검증
- findings 상세 패널 복구 후 빌드 성공
- 웹 재시작 후 API/events 연결 정상 확인
